# Boss-Fight Roadmap

Boss fights integrate established capabilities and introduce little unsupported theory. Order may adapt; scope and rigor may not be diluted.

1. B001 Expression Language: ADTs, recursion, absence/errors, laws/properties, manual parsing, precedence-aware rendering, diagnostics, deep inputs, changed syntax.
2. Unix Text Pipeline: revisit folds and productivity under streaming, bounded memory, Unicode/byte policy, and composition.
3. JSON Library: parser/renderer round trips, useful errors, depth/size, then incremental revisit.
4. Dependency Resolver: graph construction, cycles/SCCs, deterministic layers, diagnostics.
5. Miniature Compiler: scopes, name resolution, inference/checking, typed IR, optimization, VM/codegen.
6. Concurrent Job Runner: bounded work, retries, cancellation, timeout, deterministic tests, no leaks.
7. HTTP Server and Client trials: framing, partial I/O, streaming, pools, cleanup, cancellation, idempotency.
8. Mini Key-Value Store: append log, checksum, index, recovery, compaction, crash states.
9. Schema-Driven Generator: semantic model, validation, IR, deterministic printer, compile generated code, plugins.
10. Smithy Model Consumer and Generator: shapes/traits/references, projection, Haskell IR, diagnostics.
11. Focused Protocol Runtime and Small SDK: serialization, HTTP, streaming, retry classification, cancellation.
12. Capstone Cloud/Protocol Toolkit: cohesive production-oriented architecture, operations, benchmarks, docs, migration.

Each boss receives a post-boss transfer review before the next block is proposed.
