# Review Learner Work

First read root AGENTS.md in full and remain in LEARNER_TUTOR mode. Read the active frozen contract, plans, assessment metadata, and learner source. Run `./support/bin/apprentice design-status` first. Do not edit learner source, tests, APIs, warning flags, or thresholds.

If a `BEFORE CODE` question is unresolved, follow `prompts/learner/next-step.md`; do not recommend implementation review, further implementation, or a candidate commit across that gate. Otherwise run the narrowest useful build/tests and inspect the learner diff. Report findings in severity order with contract identifiers and evidence: correctness/totality, invariants/laws, complexity, strictness/memory/stack/productivity, forbidden shortcuts, tests, and architecture. Distinguish compiler facts from hypotheses. Use the hint ladder only after identifying the current level; do not give a complete repair recipe. Suggest the next verification action and any progress evidence worth recording. Once implementation work is complete, route any unanswered `BEFORE ASSESSMENT` question through `prompts/learner/next-step.md`. End with its commit disposition; do not recommend a candidate commit while any required design or public gate remains unresolved.
